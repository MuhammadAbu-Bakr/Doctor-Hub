# Doctor Hub

Doctor Hub is a Next.js healthcare portal foundation with JWT auth, role-aware protected routes, Prisma models for appointments and medical workflows, and a Render deployment blueprint.

## Local Setup

1. Install dependencies:

```bash
npm install
```

2. Copy `.env.example` to `.env` and update `DATABASE_URL` and `JWT_SECRET`.

3. Push the Prisma schema to your database:

```bash
npm run db:push
```

4. Start the app:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Supabase Database

This project can use Supabase as its PostgreSQL database.

1. Create a Supabase project.
2. Open Supabase SQL Editor.
3. Paste and run the full contents of `supabase-schema.sql`.
4. Copy your Supabase PostgreSQL URI connection string.
5. Use that value as `DATABASE_URL` in `.env` locally and in Render.

## Render Deployment

This repo includes `render.yaml` for a web service. The database should be Supabase.

1. Commit and push the repository to GitHub.
2. In Render, create a new Blueprint from the repository.
3. Add the environment variables:
   - `DATABASE_URL` is your Supabase PostgreSQL URI connection string.
   - `JWT_SECRET` is generated by Render.
   - `RESEND_API_KEY` can be left blank until email delivery is configured.
4. Deploy.

The Render build command is `npm ci && npm run render-build`, and the start command is `npm run start`. If you already ran `supabase-schema.sql`, you do not need to run `npm run db:push`.
